## ADDED Requirements

### 需求:应用列表必须展示 mythroad 目录下的 MRP 文件

应用必须在列表页扫描 `<sandbox>/mythroad/` 目录下所有 `.mrp` 后缀文件，解析其 MRPG 头部（前 240 字节），提取 appname（GBK 24 字节）、appid（uint32 LE）、version（uint32 LE）、vendor（GBK 40 字节）和 description（GBK 64 字节），以原生列表形式展示。若头部 Magic 不为 `"MRPG"` 或文件读取失败，该条目必须标记为无效并提示用户。

#### 场景:正常显示列表

- **当** 用户进入列表页，且 `<sandbox>/mythroad/` 下存在一个或多个有效的 `.mrp` 文件
- **那么** 列表必须显示每个 MRP 的应用名、版本号和开发商；无效文件不显示或标记为损坏

#### 场景:空目录

- **当** `<sandbox>/mythroad/` 下不存在任何 `.mrp` 文件
- **那么** 列表页必须显示空状态提示，并提供导入 .mrp 文件的入口

#### 场景:GBK 解码失败

- **当** 某个 MRP 头部的 appname / vendor 字段无法以 GBK 正确解码
- **那么** 该字段必须以文件名（头部 offset 16 的 filename 字段）作为 fallback 显示，不得崩溃

### 需求:点击列表项必须直接启动对应 MRP

用户点击列表中的 MRP 条目后，应用必须通过路由导航到模拟器页面，并将所选 MRP 的文件名作为参数传递。模拟器页面必须使用该文件名调用 `napi.start(filename, 'start.mr')`，绕过 `dsm_gm.mrp` 启动器。

#### 场景:成功启动 MRP

- **当** 用户点击列表中一个有效的 MRP 条目（如 `"ydqtwo.mrp"`），且该 MRP 包内存在 `start.mr`
- **那么** 应用必须导航到模拟器页面，使用 `napi.start('ydqtwo.mrp', 'start.mr')` 启动该 MRP，用户能看到模拟器画面

#### 场景:MRP 无 start.mr 入口

- **当** 用户点击的 MRP 包内不存在 `start.mr`
- **那么** 模拟器页面必须显示启动失败信息（可利用 mythroad 内部的回退逻辑尝试 `logo.ext`），且不得崩溃；用户必须能返回列表页

### 需求:无路由参数时必须保持默认启动行为

模拟器页面（Index）在未收到路由参数时，必须以 `dsm_gm.mrp` + `start.mr` 启动，保持向后兼容。

#### 场景:无参数启动

- **当** 模拟器页面通过 `router.pushUrl({ url: 'pages/Index' })` 导航（无 `params.mrp`）
- **那么** 模拟器页面必须以 `napi.start('dsm_gm.mrp', 'start.mr')` 启动，与变更前行为一致

### 需求:列表页必须提供导入 MRP 文件功能

列表页必须提供按钮或操作入口，调用系统 DocumentPicker（过滤 `.mrp` 后缀），将选中的文件拷贝到 `<sandbox>/mythroad/` 目录，拷贝完成后自动刷新列表。

#### 场景:成功导入

- **当** 用户通过 DocumentPicker 选中一个 `.mrp` 文件且拷贝成功
- **那么** 列表必须自动刷新并显示新导入的 MRP 条目，同时显示成功提示（如 toast）

#### 场景:导入同名文件

- **当** 用户导入的文件名与 mythroad 目录下已有文件同名
- **那么** 系统必须覆盖已有文件并刷新列表，不得创建重复条目

### 需求:从模拟器页返回必须回到列表页

用户在模拟器页按下返回键时，必须返回到列表页，不得退出应用或导航到其他页面。

#### 场景:返回列表

- **当** 用户在模拟器页按系统返回键
- **那么** 应用必须导航回列表页，列表状态保持不变（不重新扫描，除非有过删除/导入操作）
