## 需求
### 需求:定时器驱动
系统必须用 ArkTS 定时器替代 SDL2 定时器驱动模拟器的 timer() 回调。

#### 场景:定时器启动
- **当** ARM 代码调用 mr_timerStart(t)
- **那么** C 层必须通过 NAPI 回调通知 ArkTS 层启动一个间隔为 t 毫秒的定时器

#### 场景:定时器触发
- **当** ArkTS 定时器触发
- **那么** ArkTS 层必须调用 NAPI timer() 函数驱动模拟器执行

#### 场景:定时器停止
- **当** ARM 代码调用 mr_timerStop()
- **那么** C 层必须通过 NAPI 回调通知 ArkTS 层停止定时器

### 需求:定时器唯一性
系统在同一时间必须只运行一个定时器。

#### 场景:重复启动定时器
- **当** 在定时器运行期间再次收到 timerStart 回调
- **那么** 必须取消旧定时器并启动新定时器
